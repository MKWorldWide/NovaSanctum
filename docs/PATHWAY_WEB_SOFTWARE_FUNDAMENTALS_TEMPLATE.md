# Pathway Template: Web and Software Fundamentals

Status: Draft template (Phase 4)  
Program: NovaSanctum

## 1. Pathway Description

Web and Software Fundamentals introduces learners to core concepts needed to read, build, and maintain basic software and web artifacts using open resources.

## 2. Target Learners

1. Learners transitioning from Digital Foundations.
2. Career switchers exploring technical skills.
3. Educators/community facilitators needing software baseline fluency.

## 3. Entry Requirements

1. Digital Foundations competencies or equivalent.
2. Basic file and browser fluency.
3. Ability to install and use standard development tools.

## 4. Learning Outcomes

By pathway completion, learners should be able to:

1. Explain basic web architecture (client, server, HTTP, APIs).
2. Build simple web pages with HTML/CSS.
3. Write basic JavaScript for interactivity and data handling.
4. Use version control fundamentals (repository, commit, branch, merge).
5. Produce a small portfolio artifact with documentation.

## 5. Pathway Structure

1. Entry
   - Environment setup, terminology, and workflow orientation.
2. Foundation
   - HTML/CSS basics, command line essentials, repository fundamentals.
3. Progression
   - JavaScript basics, API calls, debugging practices, project structuring.
4. Applied Outcomes
   - Build and document a small web application or static site project.
5. Transition
   - Bridge to Data Literacy or advanced software tracks.

## 6. Example Resource Categories

1. Open web standards documentation.
2. Public tutorials for HTML/CSS/JavaScript.
3. Free version control and collaboration learning materials.
4. Open practice projects and challenge repositories.

## 7. Curation Requirements

1. Resource selection must prioritize free access and clear licensing.
2. Resources should map to explicit milestone outcomes.
3. Final pathway publication requires governance approval.
